Phoneme Map

Sometimes processing of phonological layers assumes that the annotations use the DISC phoneme set designed for the CELEX phonemic transcriptions. This set is used because each phoneme is expressed by precisely one ASCII character, including phonemes usually expressed using a digraph - e.g. affricates like /Ê§/ (which is /J/ in DISC) and diphthongs like /aÉª/ (which is /2/ in DISC)

Unisyn transcriptions use a set of phones that is greater that the set of phones available in DISC, and the transcriptions are designed to be broad-phonetic, not phonemic.

This means that using the DISC representation of the transcripts is imperfect, as there is a certain amount of loss of information when mapping Unisyn phones to DISC phonemes.

You can use ths page to customise how the Unisyn phonemes are converted into CELEX phonemes, by adding, removing, or changing phoneme mappings. Any characters in the original file which are not present in the Original column on this page are passed through to the DISC version of the pronunciation unchanged. To ignore (delete) a phoneme or character in the original representation, map it to nothing.

Some characters used in DISC for non-phonemes:

  • syllable-boundary→-
  • primary-stress→'
  • secondary-stress→"

Some ASCII characters are not used by DISC - if you require more phonemes, you could use these:

e G K M ( [ ] ! % .
