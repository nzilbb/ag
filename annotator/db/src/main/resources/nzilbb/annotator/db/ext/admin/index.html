DB Tagger

This annotator tags annotates words with data from a relational datab table loaded from a plain text file (e.g. a CSV file). The initial file must have a 'flat' structure in the sense that it's a simple list of entries with a fixed number of columns/fields. However once installed, sub-tables can be defined to represent one-to-many relationships, allowing each primary-table entry to correspond to multiple secondary-table entries.

On this page, you can add or delete tables.

Upload New Table File

The data file you supply may contain multiple fields. It might include:

  • id
  • name
  • description
  • alternative names

...or any other type data you like.

NB the text file must use ASCII or UTF-8 character encoding. If your dictionary file uses another encoding (e.g. Western or ISO-8859, you will need to re-save the file using UTF-8 (in many text editors, the character encoding is an option available when you select Save As... from the File menu).

You can upload as many data files as you like, each corresponding to a different primary table. Once you have at least one table, you can configure a word or phrase layer to lookup the resulting table entries .
