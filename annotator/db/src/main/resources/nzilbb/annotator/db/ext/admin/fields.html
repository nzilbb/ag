Fields

You can define the fields for this table here.

Field names are listed on the left. Each field has:

Type
What kind of information can be stored in this field. This can be:
  • string: short, single-line text
  • text: long multi-line text
  • richtext: long text that includes formatting, bold/italic text, links, etc.
  • integer: whole number
  • number: decimal number
  • boolean: true or false
  • url: link to a web page
  • email: email address
  • date: date (excluding time)
  • datetime: date including time of day
  • geo-location: coordinates for a geographic location
Validation
A definition for how the values for the field are constrained. What you enter here may depend on the Type specified. You can enter:
  • A list of alternatives separated by |, which will result in a dropdown list of options, e.g.
    singular|plural
    If you want to allow for there being no value specified, include a | at the beginniing of the validation, e.g.
    |singular|plural
  • For integer/number fields, you can specify the (inclusive) minimum or maximum for the value, separated by <, e.g.
    1<7
    If you want only a minimum you can omit the maximum (or vice versa) e.g.
    0<
  • For string fields, you can specify a regular expression that the value must match, e.g.
    .+@gmail\.com
Validation definitions can be up to 50 characters long.

When you make a change to the definition of a field, use the field's save button to save your changes.

You can delete a field using the field's save button . When you delete a field, all data previously saved for the field is permanantly lost, and any automated annotations that rely on that field will no longer work.

You can add a new field by pressing the 'add' button at the bottom of the list. You will be asked for the name of the field, which cannot be changed once entered. Field names can be up to 50 characters long.
